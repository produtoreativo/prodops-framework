# Capabilities

Capabilities are reusable competencies consumed by flows. They define what is done, not when or by whom.

| Capability | Purpose | Consumed by |
|---|---|---|
| [Commit Workflow](commit-workflow/README.md) | Validate, format, and version each commit | Hack, Sync, Finish |
| [Contract Management](contract-management.md) | Define and maintain verifiable contracts (OBC, BDD, OpenAPI, AsyncAPI) | Bootstrap, Hack, Sync |
| [Evidence Management](evidence-management.md) | Record, preserve, and present delivery evidence | Finish, Validate, Promote |
| [Observability](observability.md) | Emit and validate logs, metrics, and traces | Hack, Validate |
| [Reliability](reliability.md) | Define and enforce reliability requirements | Bootstrap, Hack, Finish, Promote |
